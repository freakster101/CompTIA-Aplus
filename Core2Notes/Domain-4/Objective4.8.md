# Objective 4.8 - Basics of Scripting

# Script File Types

### .bat

**Definition:** A Windows batch file used to execute command-line instructions automatically.
**Example:** A script that clears temporary files during startup.

### .ps1

**Definition:** A PowerShell script file used for Windows administration and automation.
**Example:** A script that creates new user accounts in Active Directory.

### .vbs

**Definition:** A Visual Basic Script file used to automate Windows tasks.
**Example:** A script that displays a login message to users.

### .sh

**Definition:** A shell script file used to automate tasks in Linux and Unix systems.
**Example:** A script that backs up log files every night.

### .js

**Definition:** A JavaScript file used to automate tasks in web browsers or server environments.
**Example:** A script that validates user input on a web form.

### .py

**Definition:** A Python script file used for automation, administration, and data processing.
**Example:** A script that generates daily system reports.

---

# Use Cases for Scripting

### Basic Automation

**Definition:** Using scripts to perform repetitive tasks automatically.
**Example:** Automatically deleting temporary files every week.

### Restarting Machines

**Definition:** Using scripts to reboot systems without manual intervention.
**Example:** Restarting a server after installing updates.

### Remapping Network Drives

**Definition:** Automatically reconnecting users to shared network resources.
**Example:** Mapping the Finance shared drive at user login.

### Installation of Applications

**Definition:** Automating software deployment across multiple devices.
**Example:** Installing Microsoft Office on all company laptops.

### Automated Backups

**Definition:** Running backups automatically based on a schedule.
**Example:** Backing up user files every night.

### Gathering of Information/Data

**Definition:** Collecting system or network information automatically.
**Example:** Generating a report of installed software on all PCs.

### Initiating Updates

**Definition:** Using scripts to start software or operating system updates.
**Example:** Running Windows Update on multiple workstations.

---

# Other Considerations When Using Scripts

### Unintentionally Introducing Malware

**Definition:** Running scripts from untrusted sources that contain malicious code.
**Example:** Executing a downloaded script that installs ransomware.

### Inadvertently Changing System Settings

**Definition:** Accidentally modifying system configurations through script execution.
**Example:** A script incorrectly disabling Windows Firewall.

### Browser or System Crashes Due to Mishandling of Resources

**Definition:** Poorly written scripts consuming excessive system resources.
**Example:** An infinite loop causing a computer to become unresponsive.
