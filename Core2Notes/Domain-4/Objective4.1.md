# Objective 4.1 - Documentation and Support Systems Information Management

## Ticketing Systems

### User Information

**Definition:** Information that identifies the user experiencing the issue.
**Example:** John Smith, Finance Department, [john.smith@company.com](mailto:john.smith@company.com).

### Device Information

**Definition:** Information that identifies the affected device.
**Example:** Dell Latitude 5440, Asset Tag LAP-1023.

### Description of Issues

**Definition:** A summary of the problem reported by the user.
**Example:** User cannot access Outlook email.

### Categories

**Definition:** Classification used to organize support tickets.
**Example:** Software → Microsoft Outlook.

### Severity

**Definition:** The level of impact an issue has on business operations.
**Example:** Email outage affecting all employees = Critical.

### Escalation Levels

**Definition:** Defined levels of support for handling complex issues.
**Example:** A Tier 1 technician escalates a server issue to Tier 2.

### Clear, Concise Written Communication

**Definition:** Documentation written in a professional and easy-to-understand manner.
**Example:** "Printer queue cleared and printing restored."

### Issue Description

**Definition:** The original problem recorded in the ticket.
**Example:** User unable to connect to VPN.

### Progress Notes

**Definition:** Documentation of troubleshooting steps taken.
**Example:** Reset user password and tested login.

### Issue Resolution

**Definition:** Documentation of the final solution.
**Example:** Reinstalled Outlook and verified functionality.

# Asset Management

### Inventory Lists

**Definition:** Records of organizational hardware and software assets.
**Example:** Spreadsheet containing all company laptops.

### Configuration Management Database (CMDB)

**Definition:** A database containing detailed information about IT assets and their relationships.
**Example:** CMDB showing a server's hardware, software, and network details.

### Asset Tags and IDs

**Definition:** Unique identifiers assigned to company assets.
**Example:** Asset Tag: PC-1001.

### Procurement Life Cycle

**Definition:** The process of acquiring, using, maintaining, and retiring assets.
**Example:** Purchasing a laptop, assigning it to a user, and replacing it after five years.

### Warranty and Licensing

**Definition:** Documentation tracking hardware warranties and software licenses.
**Example:** Microsoft Office license assigned to a user.

### Assigned Users

**Definition:** Records showing which user is responsible for an asset.
**Example:** Laptop LAP-1001 assigned to Sarah Johnson.

# Types of Documents

### Incident Reports

**Definition:** Documents describing security incidents, outages, or major problems.
**Example:** Report documenting a malware infection on a workstation.

### Standard Operating Procedures (SOPs)

**Definition:** Step-by-step instructions for performing routine tasks.
**Example:** Password reset procedure.

### Software Package Custom Installation Procedure

**Definition:** Instructions for installing and configuring approved software.
**Example:** Company-specific Microsoft Office installation guide.

### New User/Onboarding Setup Checklist

**Definition:** A checklist used when preparing accounts and devices for new employees.
**Example:** Create email account, configure laptop, assign permissions.

### User Off-Boarding Checklist

**Definition:** A checklist used when removing access for departing employees.
**Example:** Disable account and collect company laptop.

### Service-Level Agreements (SLAs)

**Definition:** Agreements that define expected service performance and response times.
**Example:** IT must respond to support tickets within four hours.

#### Internal SLA

**Definition:** An SLA between departments within the same organization.
**Example:** IT support responding to employee tickets within four hours.

#### External/Third-Party SLA

**Definition:** An SLA between an organization and an outside vendor.
**Example:** Internet provider guarantees 99.9% uptime.

### Knowledge Base/Articles

**Definition:** Documents containing solutions and information for common issues.
**Example:** Article explaining how to configure Outlook.
