Objective 4.2 - Cloud Computing Concepts

Common Cloud Models

Private Cloud

Definition: A cloud environment dedicated to a single organization.
Example: A company hosting its own cloud infrastructure for employees.

Public Cloud

Definition: A cloud environment owned and operated by a third-party provider and available to the public.
Example: Using Amazon Web Services (AWS) for cloud storage.

Hybrid Cloud

Definition: A cloud model that combines private and public cloud resources.
Example: Storing sensitive data in a private cloud while using a public cloud for web hosting.

Community Cloud

Definition: A cloud environment shared by organizations with similar requirements.
Example: Several hospitals sharing a healthcare-focused cloud platform.

Infrastructure as a Service (IaaS)

Definition: A cloud service model that provides virtualized computing resources.
Example: Creating virtual servers in Microsoft Azure.

Software as a Service (SaaS)

Definition: A cloud service model that delivers software applications over the internet.
Example: Using Microsoft 365 through a web browser.

Platform as a Service (PaaS)

Definition: A cloud service model that provides a platform for application development and deployment.
Example: Developing an application using Google App Engine.

Cloud Characteristics

Shared Resources

Definition: Cloud resources that are used by multiple customers.
Example: Multiple organizations using the same cloud data center infrastructure.

Dedicated Resources

Definition: Cloud resources reserved for a single customer.
Example: A dedicated cloud server assigned to one company.

Metered Utilization

Definition: A billing model where customers pay based on resource consumption.
Example: Paying only for the storage space used in the cloud.

Ingress

Definition: Data entering a cloud environment.
Example: Uploading files from a local computer to cloud storage.

Egress

Definition: Data leaving a cloud environment.
Example: Downloading files from cloud storage to a laptop.

Elasticity

Definition: The ability to automatically increase or decrease resources as demand changes.
Example: A website automatically adding servers during heavy traffic.

Availability

Definition: The degree to which cloud services remain accessible and operational.
Example: A cloud provider offering 99.99% uptime.

File Synchronization

Definition: The process of keeping files consistent across multiple devices and locations.
Example: A document edited on a laptop automatically updating on a smartphone.

Multitenancy

Definition: A cloud architecture where multiple customers share the same infrastructure.
Example: Multiple businesses using the same SaaS application instance.
