Objective 4.1 - Virtualization Concepts

Purpose of Virtual Machines

Virtual Machine (VM)

Definition: A software-based computer that runs an operating system and applications independently of the host system.
Example: Running Windows 11 inside VirtualBox on a Linux computer.

Sandbox

Definition: An isolated environment used to safely test software or files.
Example: Opening a suspicious application inside a VM before using it on a production system.

Test Development

Definition: Using virtual machines to develop and test software without affecting production systems.
Example: Testing a new application in a VM before deployment.

Application Virtualization

Definition: Running applications in an isolated virtual environment without directly installing them on the host OS.
Example: Running a virtualized version of Microsoft Office.

Legacy Software/OS

Definition: Using virtualization to run older software or operating systems on modern hardware.
Example: Running Windows XP in a VM to support an older application.

Cross-Platform Virtualization

Definition: Running an operating system that differs from the host operating system.
Example: Running Linux on a Windows computer using a VM.

---

Requirements

Security

Definition: Measures used to protect virtual machines and virtualized environments.
Example: Applying security updates to a virtual machine.

Network

Definition: Connectivity resources required for communication between virtual and physical systems.
Example: Assigning a virtual machine to a virtual network.

Storage

Definition: Disk space used to store virtual machines and their data.
Example: Allocating 100GB of storage to a VM.

---

Desktop Virtualization

Desktop Virtualization

Definition: A technology that hosts desktop environments on centralized servers.
Example: Employees accessing their desktops remotely from thin clients.

Virtual Desktop Infrastructure (VDI)

Definition: A desktop virtualization solution that delivers virtual desktops from a centralized server.
Example: Users logging into virtual desktops hosted in a data center.

---

Containers

Container

Definition: A lightweight virtualization technology that packages applications and their dependencies.
Example: Running a web application inside a Docker container.

---

Hypervisors

Hypervisor

Definition: Software or firmware that creates and manages virtual machines.
Example: VMware ESXi managing multiple virtual machines.

Type 1 Hypervisor

Definition: A hypervisor that runs directly on physical hardware.
Example: VMware ESXi installed directly on a server.

Type 2 Hypervisor

Definition: A hypervisor that runs on top of an operating system.
Example: Oracle VirtualBox running on Windows 11.
